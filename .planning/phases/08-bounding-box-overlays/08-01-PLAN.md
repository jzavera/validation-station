# Plan 08-01: Bounding Box Overlays

## Goal
Build the BoundingBoxOverlay component and wire it into PdfViewer and ImageViewer so that the active field's extraction region is visually highlighted on the document.

## Tasks

1. **Create `src/components/BoundingBoxOverlay.tsx`**
   - Props: `activeField: Field | null`, `pageNumber: number`
   - If no active field or field's bounding box is on a different page, render nothing
   - Prioritize token-level bounding boxes; fall back to field-level boundingBox
   - Render each box as an absolutely positioned div with percentage-based coordinates
   - Wrap in pointer-events-none container

2. **Update `src/components/DocumentViewer.tsx`**
   - Derive `activeField` from state using `activeFieldId` lookup
   - Pass `activeField` to PdfViewer and ImageViewer

3. **Update `src/components/PdfViewer.tsx`**
   - Accept `activeField` prop
   - Render `<BoundingBoxOverlay>` inside each page's relative wrapper

4. **Update `src/components/ImageViewer.tsx`**
   - Accept `activeField` prop
   - Render `<BoundingBoxOverlay>` inside the image's relative wrapper

## Requirements Covered
- BBOX-01: Token-level overlays on active field
- BBOX-02: Field-level fallback when no tokens
- BBOX-03: Normalized 0-1 coordinates with percentage CSS
- BBOX-04: Correct alignment during zoom
- BBOX-05: Only active field overlays rendered

## Verification
- `npx tsc --noEmit` passes
- `npm run build` succeeds
- Active field shows blue highlight overlays on the document
- Zooming preserves overlay alignment
